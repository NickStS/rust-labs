1. Навчитися використовувати Rust документацію та документацію залежностей локально. Дослідити методи локального кешування залежностей для використання у проектах коли не має доступу до інтернету. Дослідити методи кешування крейтів.

Локальна Rust документація встановлюється разом з тулчейном через rustup component add rust-docs. Її можна відкрити командою rustup doc — вона запускає браузер з вмістом стандартної бібліотеки, The Book, Rust by Example та інших офіційних посібників. Для конкретних типів існує rustup doc --std std::vec::Vec. Документація залежностей проекту генерується командою cargo doc — вона збирає HTML з doc-коментарів усіх крейтів у залежностях і складає у target/doc. Прапорець --open одразу відкриває її у браузері, а --no-deps генерує лише для поточного крейту без залежностей. Для роботи без інтернету Cargo має кілька механізмів. Усі завантажені крейти кешуються у ~/.cargo/registry/cache та ~/.cargo/registry/src — звідти Cargo бере їх при повторному використанні. Для повністю офлайн режиму використовують прапорець --offline що забороняє Cargo звертатися до мережі. cargo vendor створює локальну копію усіх залежностей у директорії vendor/ та конфігурацію для перенаправлення Cargo на цю директорію — після цього проект можна збирати без інтернету. cargo fetch заздалегідь завантажує всі залежності у локальний кеш. Для команд з спільним кешем використовують dragoman або sccache що кешують також скомпільовані артефакти.

2. Дослідити шляхи документування Rust проектів: як створити власну документацію, який синтаксис написання Rust документації.

Документація у Rust пишеться спеціальними коментарями: /// для документування елементу що йде далі (функції, структури, поля), //! для документування модуля або крейту зсередини. Підтримується синтаксис Markdown — заголовки, списки, посилання, таблиці, блоки коду. Блоки коду на Rust автоматично виконуються як doc-tests при cargo test, що дозволяє підтримувати приклади актуальними. Існують стандартні секції: # Examples для прикладів використання, # Errors для опису можливих помилок у функціях що повертають Result, # Panics для умов паніки, # Safety для unsafe-функцій з описом інваріантів які повинен підтримати викликач. Посилання на інші елементи робляться через інтра-док лінки [Type] або [module::function] — Rustdoc автоматично перетворить їх на робочі посилання. Команда cargo doc --open збирає HTML з усіх цих коментарів і відкриває у браузері. Для розширеного контролю є атрибут #![doc = "..."] що приймає включений з файлу markdown через #![doc = include_str!("../README.md")]. Лінти missing_docs, missing_crate_level_docs, missing_panics_doc та missing_errors_doc дозволяють вимагати документацію на рівні компіляції.

3. Навчитися публікувати Rust бібліотеки на crates.io. Описати обмеження та вимоги до крейтів перед публікацією. Розглянути можливості власних реєстрів крейтів.

Публікація на crates.io починається з реєстрації акаунту через GitHub та отримання API-токена. Токен зберігається локально командою cargo login. Перед публікацією Cargo.toml має містити певні поля: name, version, edition, description, license або license-file, repository або homepage, keywords та categories. Без license та description Cargo не дозволить публікацію. Назва крейту має бути унікальною на crates.io — повторне використання навіть видаленої назви заборонено. Версія повинна відповідати semver — мажорна для breaking changes, мінорна для нової функціональності, патч для виправлень. Команда cargo publish збирає крейт, виконує тести, перевіряє що документація компілюється і завантажує архів на crates.io. Опубліковану версію не можна змінити — лише позначити yanked щоб нові проекти її не використовували. Для приватних реєстрів існують декілька опцій. cloudsmith, jfrog artifactory, gitlab package registry мають підтримку cargo registry API. Для self-hosted рішень є alexandrie та kellnr — open-source реєстри що повністю сумісні з cargo. Налаштування власного реєстру у Cargo робиться через .cargo/config.toml у секції [registries] з URL до index repository. Залежності з приватного реєстру вказуються через registry = "name" у Cargo.toml.
