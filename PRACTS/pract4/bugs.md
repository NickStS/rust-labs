Bug #1: JsonIndex::open та SqliteIndex::open панікують замість повернення помилки
Пріоритет: High
Опис: Методи open() використовують expect() та unwrap(), що призводить до паніки при помилці відкриття файлу або бази даних. Бібліотечний код не повинен панікувати — має повертати Result.
Статус: Fixed

Bug #2: SQLite get() повертає всі записи замість фільтрації на рівні SQL
Пріоритет: Medium
Опис: Метод get() у SqliteIndex виконує SELECT без WHERE по тегах, завантажує всі записи в пам'ять і фільтрує вже в Rust. При великій кількості записів це неефективно. Потрібно фільтрувати на рівні SQL-запиту.
Статус: Fixed

Bug #3: Немає команди для видалення записів з індексу
Пріоритет: Low
Опис: CLI не має команди remove для видалення файлу з індексу. Користувач не може прибрати помилково доданий запис без ручного редагування JSON або SQL.
Статус: Fixed
