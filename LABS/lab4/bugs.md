Bug #1: Вихідні файли втрачають оригінальне ім'я
Severity: Major
Priority: High
Опис: При обробці кожне зображення зберігається як 0.png, 1.png, 2.png — лише за порядковим номером у списку. Якщо обробляється багато файлів, неможливо зрозуміти який вихідний файл відповідає якому вхідному. Імя оригінального файлу або URL втрачається повністю.
Кроки відтворення:
1. Створити images.txt зі списком кількох URL зображень
2. Запустити lab4 --files images.txt --resize 200x200
3. Подивитись на вихідну директорію — всі файли названі лише за індексом
Очікувано: ім'я має містити частину оригінального імені файлу для ідентифікації
Статус: Fixed
Виправлення: додано функцію derive_name яка формує ім'я виду {index}_{stem}.png зі stem від оригінального шляху або URL

Bug #2: Зависання програми на повільних HTTP запитах
Severity: Major
Priority: Medium
Опис: reqwest::blocking::get() використовується без таймауту. Якщо сервер не відповідає або зображення дуже велике, програма може зависнути на невизначений час, блокуючи обробку всіх наступних рядків у списку.
Кроки відтворення:
1. Додати у images.txt URL що повільно відповідає або не відповідає взагалі
2. Запустити програму
3. Програма зависає без можливості переривання іншою помилкою
Очікувано: запит має завершитися з помилкою через розумний таймаут
Статус: Fixed
Виправлення: додано http_client() з timeout=30s та connect_timeout=10s, замість прямого виклику reqwest::blocking::get

Bug #3: Спотворення зображень при resize
Severity: Minor
Priority: Medium
Опис: Завжди використовується resize_exact, який ігнорує співвідношення сторін. Якщо вихідне зображення має пропорції 16:9 а цільовий розмір 200x200, зображення буде розтягнуте і виглядатиме спотворено. Не має опції для збереження пропорцій.
Кроки відтворення:
1. Запустити з зображенням широкого формату (наприклад 1920x1080)
2. --resize 200x200
3. Результат: квадратне зображення зі стиснутим по горизонталі контентом
Очікувано: повинна бути опція для збереження пропорцій (вписати в межі)
Статус: Fixed
Виправлення: додано CLI флаг --keep-aspect, при якому використовується image::resize замість resize_exact
