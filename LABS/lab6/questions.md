1. Що таке атоміки?

Атоміки — це спеціальні типи даних які гарантують неподільність операцій читання, запису та модифікації навіть при одночасному доступі з кількох потоків. Жоден потік не може побачити частково оновлене значення. У Rust це AtomicBool, AtomicUsize, AtomicI32 та інші у std::sync::atomic. Вони реалізовані на основі спеціальних інструкцій процесора (LOCK на x86, LDREX/STREX на ARM) і не потребують блокувань. Атоміки — це базова цеглинка для побудови складніших примітивів синхронізації як Mutex, RwLock, Arc.

2. Що таке memory ordering?

Memory ordering — це гарантії що задаються при атомарній операції про те як вона співвідноситься з іншими операціями памяті у термінах видимості та порядку виконання. Сучасні процесори та компілятори можуть переставляти операції памяті заради продуктивності. Memory ordering каже компілятору і процесору які перестановки заборонені. Наприклад, якщо потік A пише дані а потім встановлює прапорець, потік B який бачить прапорець у true повинен також бачити записані дані — це досягається правильним ordering у обох потоках.

3. Чим відрізняються різні memory ordering?

Relaxed — найслабша гарантія, операція є атомарною але не синхронізує жодних інших операцій памяті. Підходить для простих лічильників де порядок не важливий.

Acquire — гарантує що всі читання та записи що йдуть ПІСЛЯ цієї операції в коді не будуть переставлені перед нею. Використовується при читанні (load) та забезпечує що потік побачить всі зміни які зробив інший потік до Release у тій самій локації.

Release — гарантує що всі читання та записи що йдуть ПЕРЕД цією операцією в коді не будуть переставлені після неї. Використовується при записі (store) для публікації змін іншим потокам.

AcqRel — комбінація Acquire і Release для read-modify-write операцій (compare_exchange, fetch_add).

SeqCst (Sequential Consistency) — найсильніша гарантія: всі SeqCst операції утворюють єдиний глобальний порядок який бачать усі потоки однаково. Найдорожчий варіант але найпростіший для розуміння. Використовувати коли точно потрібен глобальний порядок між кількома локаціями.
