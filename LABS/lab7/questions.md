1. Що таке асинхронний рантайм? Що таке tokio?

Асинхронний рантайм — це бібліотека що виконує асинхронні задачі (Future). Він керує пулом потоків, реактором який реєструє IO-події через системні API (epoll, kqueue, IOCP), планує запуск задач коли їх waker отримує сигнал готовності, і надає примітиви для асинхронного IO, таймерів, синхронізації. У Rust трейт Future описує що таке асинхронна задача але не як її виконувати — це робота рантайму. Rust навмисно не має вбудованого рантайму, дозволяючи обирати під задачу.

tokio — найпопулярніший асинхронний рантайм для Rust. Він надає multi-threaded scheduler з work-stealing, асинхронні версії стандартних API (tokio::fs, tokio::net, tokio::io), таймери, синхронізаційні примітиви (Mutex, RwLock, Semaphore, channels), а також інтеграцію з більшістю екосистеми (reqwest, hyper, sqlx тощо). Завдяки макросу #[tokio::main] асинхронну програму можна запустити з звичайного main.

2. Що таке blocking thread pool у tokio? Для чого він використовується?

Tokio має два пули потоків. Worker threads — невеликий пул (за замовчуванням за кількістю ядер) на якому виконуються асинхронні задачі. Blocking thread pool — окремий пул що зростає за потребою (за замовчуванням до 512 потоків) для виконання блокувальних або CPU-bound операцій.

Він використовується через tokio::task::spawn_blocking. Туди слід виносити: синхронні системні виклики що не мають асинхронних аналогів, важкі обчислення (декодування зображень, шифрування, парсинг великих файлів), використання блокувальних бібліотек. Якщо такий код запустити прямо у async-функції, він заблокує worker-потік і інші асинхронні задачі що мали б на ньому виконуватися застрягнуть. Перенесення на blocking pool зберігає основні worker-потоки вільними для обслуговування IO-подій.
