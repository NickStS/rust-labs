1. Що таке поліморфізм? Які обмеження має динамічний поліморфізм? Коли неможливо використовувати динамічний поліморфізм?

Поліморфізм — це здатність коду працювати з різними типами через єдиний інтерфейс. У Rust є два види поліморфізму: статичний (мономорфізація через generics) та динамічний (через trait objects, тобто dyn Trait).

Динамічний поліморфізм реалізується через fat pointer: вказівник на дані + вказівник на vtable з адресами методів конкретного типу. Вибір методу відбувається у runtime, тому компілятор не може інлайнити виклики — це основне обмеження з точки зору продуктивності.

Динамічний поліморфізм неможливо використовувати якщо trait не є object-safe. Trait не є object-safe коли: метод повертає Self, метод має generic-параметри, або метод приймає Self за значенням (крім receiver).

2. Що таке таблиця віртуалізації? Як вона працює?

Таблиця віртуалізації (vtable) — це статична таблиця вказівників на функції, яку компілятор генерує для кожної пари (тип, trait). Кожен елемент таблиці — адреса конкретної реалізації відповідного методу для даного типу.

Коли створюється trait object (dyn Trait), Rust будує fat pointer: перша частина вказує на самі дані, друга — на vtable. При виклику методу через dyn Trait відбувається indirect call: зчитується адреса методу з vtable і викликається функція за цією адресою. Саме через цей indirect call динамічний поліморфізм повільніший за статичний, але дозволяє зберігати різнотипні значення в одній колекції і вирішувати вибір реалізації у runtime.
